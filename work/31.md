`new URL()` 是 JavaScript 和 Node.js 中用于解析和操作 URL 的内置构造函数，遵循 WHATWG URL 标准。它提供了一种高效、安全的方式处理 URL 的各个组成部分（如协议、主机名、查询参数等）。以下从核心功能、使用方法和实际场景展开详解：

---

### 📌 一、基本语法与创建
```javascript
// 解析绝对 URL
const url1 = new URL('https://example.com/path?name=John#section');
console.log(url1.href); // https://example.com/path?name=John#section

// 解析相对 URL（需 base 参数）
const baseUrl = 'https://example.com/base/';
const url2 = new URL('subpath', baseUrl); 
console.log(url2.href); // https://example.com/base/subpath
```
- **参数说明**：
  - `input`：需解析的 URL 字符串（绝对或相对路径）。
  - `base`（可选）：当 `input` 为相对路径时，必须提供基础 URL。

---

### 🧩 二、核心属性与方法
#### 1. 属性解析（以 `https://user:pass@example.com:8080/path?q=test#hash` 为例）
| **属性**       | **返回值**                     | **说明**                     |
|----------------|-------------------------------|-----------------------------|
| `protocol`     | `https:`                      | 协议（含冒号）              |
| `username`     | `user`                        | 用户名                      |
| `password`     | `pass`                        | 密码                        |
| `host`         | `example.com:8080`            | 主机名 + 端口              |
| `hostname`     | `example.com`                 | 主机名（不含端口）          |
| `port`         | `8080`                        | 端口号                      |
| `pathname`     | `/path`                       | 路径部分                    |
| `search`       | `?q=test`                     | 查询字符串（含 `?`）        |
| `searchParams` | `URLSearchParams` 对象        | 用于操作查询参数（见下文）  |
| `hash`         | `#hash`                       | 哈希值（含 `#`）            |
| `origin`       | `https://example.com:8080`    | 源（协议+主机+端口）        |

#### 2. 操作查询参数 (`searchParams`)
```javascript
const url = new URL('https://example.com?name=John&age=30');

// 获取参数
url.searchParams.get('name'); // "John"

// 添加参数
url.searchParams.append('lang', 'en'); // URL 变为 ?name=John&age=30&lang=en

// 修改参数
url.searchParams.set('age', 25); // 替换 age=30 为 age=25

// 删除参数
url.searchParams.delete('name'); // 移除 name 参数

// 自动编码特殊字符
url.searchParams.set('key', '空格!@'); 
console.log(url.search); // ?key=%E7%A9%BA%E6%A0%BC%21%40
```
- **特性**：  
  - 自动处理编码（如空格→`%20`、`!`→`%21`）。
  - 支持多值参数（如 `?id=1&id=2`），用 `getAll()` 获取所有值。

### ⚙️ 三、关键使用场景
1. **动态路径拼接（前端框架）**  
   在 Vite 等工具中，结合 `import.meta.url` 安全引用静态资源：
   ```javascript
   const imagePath = new URL('./assets/logo.png', import.meta.url).href;
   ```
   - ✅ 优势：自动处理构建后的文件哈希、跨平台路径兼容。

2. **域名国际化处理**  
   自动将 Unicode 域名（如 `測試.com`）转换为 Punycode 格式（`xn--g6w251d.com`）：
   ```javascript
   const url = new URL('https://測試.com');
   console.log(url.hostname); // xn--g6w251d.com 
   ```

3. **替换传统 `url.parse()`（Node.js）**  
   相比 Node.js 旧版 `url.parse()`，`new URL()` 更符合标准且支持 `searchParams`：
   ```javascript
   // 旧方法（已废弃）
   const oldUrl = require('url').parse('https://example.com?q=1');

   // 新方法（推荐）
   const newUrl = new URL('https://example.com?q=1');
   ```

---

### ⚠️ 四、注意事项
1. **错误处理**  
   无效 URL 会抛出 `TypeError`，需用 `try/catch` 捕获：
   ```javascript
   try {
     const url = new URL('invalid-url');
   } catch (err) {
     console.error('Invalid URL'); // 捕获异常
   }
   ```

2. **Node.js 特殊行为**  
   - 若未启用 ICU 编译，Unicode 域名不会自动转换。
   - 避免与 `url` 模块变量冲突：
     ```javascript
     // ❌ 错误：覆盖全局 URL
     const URL = require('url'); 

     // ✅ 正确：重命名模块
     const urlModule = require('url');
     const myUrl = new URL('https://example.com'); // 仍可用全局 URL
     ```

3. **相对路径解析规则**  
   | **Input**           | **Base**               | **Result**                  |
   |---------------------|------------------------|----------------------------|
   | `/admin`            | `https://site.com`     | `https://site.com/admin`   |
   | `http:path`         | `https://site.com`     | `http://site.com/path`     |
   | `https:path`        | `https://site.com`     | `https://site.com/path`    |
   | `foo:bar`           | `https://site.com`     | `foo:bar`（绝对协议不变） |

---

### 💎 总结
- **核心用途**：安全解析 URL、操作查询参数、处理编码/解码。
- **最佳实践**：  
  - 前端：用 `new URL(relativePath, import.meta.url)` 引用资源（Vite 等工具）。
  - 后端：在 Node.js 中替代 `url.parse()`，注意模块冲突。
- **优势**：标准化、自动编码、跨平台一致性，是现代化 URL 处理的首选方案。
